1. Как вы можете описать процесс распространения событий в JavaScript (Event Propagation)?
Расскажите о его основных фазах.
Какие есть способы применения знаний о фазах распространения событий на практике?

Event Propagation - это механизм, по которому событие проходит через DOM-дерево

Процесс распространения события состоит из трёх фаз:
    1) Событие идёт сверху вниз от корня документа до целевого элемента.
    2) Событие достигает целевого элемента.
    3) Событие поднимается обратно от целевого элемента к корню.

Практическое применение:
    Остановка всплытия через event.stopPropagation(), чтобы событие не триггерилось на родителях.
    Делегирование событий: вешаем один обработчик на родителя, а не на каждый дочерний элемент

2. Расскажите, что такое Promise в JavaScript, как он работает, какие основные состояния у него бывают?
Каким еще способом можно обработать асинхронный код?
Какую роль в асинхронности играет событийный цикл (Event Loop)?

Promise — это объект, который представляет результат асинхронной операции. Он создается через new Promise(), принимает ф-ии resolve, reject. После можно обрабатывать через .then(), .catch(), .finally()
Основные состояния:
    1) pending - ожидание
    2) fulfilled - успешно выполнено
    3) rejected - выполнено с ошибкой

Другие способы:
    1) async\await - позволяет писать ассинхронный код, как синхронный
    2) Callback-функции

Event Loop управляет выполнением кода.

3. Что такое ООП, и какие его ключевые принципы вы могли бы описать?
При объяснении приводите примеры.
Максимально подробно расскажите о том, как ООП реализовано в JavaScript.

Объектно ориентированное программирование (ООП) - подход, где программа строится из объектов, которые взаимодействуют друг с другом.
Ключевые принципы:
    1) Инкапсуляция - скрытие внутренней логики объекта. Пример: класс с приватными полями
    2) Наследование - класс может наследовать свойства и методы другого. Пример: class Animal(), class Dog extends Animal(0
    3) Полиморфизм - разные объекты могут использовать методы по-своему. Пример: метод makeSound() у класса Animal и переопределенный в Dog

В JavaScript ООП реализовано на основе прототипов, а не классов.
    1. Прототипное наследование
    Каждый объект имеет скрытое свойство [[Prototype]], ссылающееся на другой объект (прототип). При обращении к свойству или методу, если их нет в самом объекте, поиск идет по цепочке прототипов.
    2. Конструкторы и new
    Функции-конструкторы создают объекты, а new создает пустой объект, привязывает его к this в конструкторе и устанавливает прототипом Animal.prototype.

4. Опишите максимально детально, как браузер обрабатывает URL, начиная с момента его ввода в адресную строку и заканчивая отображением страницы.
Какие технологии позволяют ускорить процесс?
Какие проблемы (безопасность, междоменное взаимодействие) могут возникать в процессе?

Обработка URL браузером:
    1) Парсинг URL: Браузер разбирает введённый адрес на протокол, домен, путь и т.д.
    2) DNS-запрос: Поиск IP-адреса домена через DNS-сервер.
    3) Установка TCP-соединения: «Рукопожатие» с сервером (HTTPS добавляет этап шифрования).
    4) HTTP-запрос: Браузер отправляет запрос на сервер.
    5) Обработка ответа: Сервер возвращает HTML, CSS, JS и другие ресурсы.
    6) Рендеринг:
        Построение DOM и CSSOM.
        Формирование дерева рендеринга.
        Отображение на экране

Технологии для ускорения:
    1) Кэширование.
    2) Предзагрузка ресурсов.
    3) Минификация и сжатие файлов.

Проблемы:
    Безопасность: XSS-атаки, CSRF, утечки данных.
    Междоменное взаимодействие: CORS-политики блокируют запросы к другим доменам без разрешения.
